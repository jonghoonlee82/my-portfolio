멍냥 
반려동물 용품을 판매하는 쇼핑몰 웹사이트로,
HTML, CSS, JavaScript, jQuery를 활용해 프론트엔드 구조와 UI를 구현했습니다.

회원가입 시 주소 입력 편의를 위해 다음 주소 API를 연동했고,
사용자가 직관적으로 상품을 탐색할 수 있도록 다양한 인터랙션을 적용했습니다.

실제 쇼핑몰 흐름을 체험할 수 있는 형태로 설계하였습니다.



상황
좌측 Sticky 카테고리 메뉴에 마우스를 올리면 아이콘 이미지가 흰색 버전으로 변경되는 hover 기능을 구현하였다. 하지만 hover 시 이미지가 순간적으로 깜빡이거나 부드럽지 않게 전환되는 문제가 발생하였다.
문제 원인을 찾아본 결과, 이미지 src를 직접 교체하는 방식으로 처리할 경우 브라우저가 이미지를 다시 로딩하면서 깜빡임이 발생할 수 있었다.

.sticky_img img {
  transition: opacity 100ms ease;
}

해결

이미지를 교체하는 대신 opacity 기반 transition을 적용하여 자연스럽게 전환되도록 개선하였다.
또한 will-change: opacity 속성을 사용하여 브라우저 렌더링 성능을 최적화하였다.

.sticky_img img {
  display: block;
  transition: opacity 100ms ease;
  will-change: opacity;
}




상황

정렬 옵션을 radio + label 구조로 구현하였다.

문제

일부 환경에서 label 클릭 시 radio change 이벤트가 정상적으로 발생하지 않아
정렬이 동작하지 않는 문제가 발생하였다.

해결

label 클릭 시 직접 input을 체크하고 change 이벤트를 강제로 발생시키도록 처리하였다.

label.addEventListener('click', () => {
  const input = document.getElementById(forId);
  if (input) {
    input.checked = true;
    input.dispatchEvent(new Event('change', { bubbles: true }));
  }
});
결과

모든 환경에서 label 클릭만으로도 안정적으로 정렬 기능이 동작하도록 개선하였다.




상황

상품 리스트에서 카테고리 필터(dog)와 정렬(detail)을 동시에 적용하는 기능을 구현하였다.
하지만
필터와 정렬이 각각 독립적으로 동작하면서
사용자가 필터를 변경할 때 정렬 상태가 유지되지 않거나,
정렬을 변경할 때 필터가 초기화되는 문제가 발생하였다.

필터와 정렬 상태를 각각 이벤트에서 개별적으로 처리하면서
전체 상태를 통합 관리하지 못한 구조였다.

해결

currentDog, currentDetail 상태값을 분리하여 관리하고,
모든 변경 시 renderItems() 하나의 함수에서 필터 + 정렬을 동시에 처리하도록 구조를 개선하였다.

let currentDog = 'all';
let currentDetail = 'rank';

function renderItems() {
  let visibleItems = Array.from(items).filter(item => {
    if (currentDog === 'all') return true;
    return item.dataset.type === currentDog;
  });

  visibleItems.sort((a, b) => {
    return toNum(a.dataset.rank) - toNum(b.dataset.rank);
  });
}





상황

상품 데이터를 HTML의 data-* 속성으로 관리하고 이를 기준으로 정렬 기능을 구현하였다.

문제

dataset으로 가져온 값이 문자열(string)로 처리되어
숫자 정렬이 정상적으로 동작하지 않는 문제가 발생하였다.

해결

공통 변환 함수 toNum()을 만들어 모든 정렬에서 숫자로 변환하도록 처리하였다.

function toNum(val) {
  const n = Number(val);
  return Number.isFinite(n) ? n : 0;
}

👉 반복되는 코드를 줄이고 유지보수성을 향상시켰습니다.





문제 상황

스크롤 위치에 따라 현재 섹션을 표시하는 과정에서
활성화 타이밍이 부정확하거나 페이지 하단에서 메뉴가 정상적으로 동작하지 않는 문제가 발생했습니다.

문제 코드 (핵심 부분)
if (scroll >= $(sec).offset().top - 200) {
  $(".fix_menu > div").removeClass("active");
  $(".fix_menu > div").eq(i).addClass("active");
}


해결 방법

scrollTop, windowHeight, documentHeight 값을 활용하여
각 섹션의 위치를 기준으로 active 상태를 제어하고,

페이지 최하단 도달 시 마지막 메뉴가 활성화되도록 예외 처리를 추가했습니다.

👉 스크롤 위치에 따른 메뉴 동기화를 정확하게 개선했습니다.

if (scroll + windowHeight >= documentHeight - 10) {
  $(".fix_menu > div").removeClass("active");
  $(".fix_menu > div").eq(sections.length - 1).addClass("active");
}
